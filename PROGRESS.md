## How to get a working environment

```bash
bash scripts/setup_env.sh      # one-time: mamba env + Flutter SDK + Android SDK into .tooling/
source scripts/activate.sh     # every shell: puts flutter/dart/java/sdkmanager on PATH
flutter doctor                 # Flutter + Android toolchain should be green
```

Everything project-specific is isolated:
- `environment.yml` → mamba env `zany-test-prep` (JDK 17 + python content tooling).
- `.tooling/` (git-ignored) → Flutter 3.35.5 SDK + Android SDK 35, installed by `scripts/setup_env.sh`.

## Architecture (quick map)

- `lib/app/` — `AppController` (Riverpod AsyncNotifier; bootstrap/lessons/review/profile/reset),
  `router.dart` (GoRouter), `app.dart` (themed MaterialApp.router + splash).
- `lib/domain/models/` — exam/skill/lesson/question/content_bundle + profile/progress/gamification.
- `lib/domain/services/` — answer_checker, leveling (XP+level), streak, mastery, unlock, review, badges, game_service.
- `lib/data/` — `KeyValueStore` (shared_preferences + memory), content & progress repositories.
- `lib/design/` — theme + shared widgets. `lib/features/` — onboarding, home, lessons, review, progress, achievements, settings.
- Canonical content in `content/exams/sat/`; runtime bundle in `assets/content/sat.bundle.json`.

## Notes / decisions

- Content is generated by deterministic Python generators (math computed; RW
  rule-based/combinatorial; authored reading passages) → guaranteed-correct, auditable.
- Persistence is JSON in `shared_preferences` behind a `KeyValueStore` interface
  (easy to fake in tests; swap to Drift/Isar later if needed).
- Widget tests inject the content bundle synchronously and use an in-memory store, so
  the suite is deterministic and headless (no device needed for CI).
