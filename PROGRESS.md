## How to get a working environment

```bash
bash scripts/setup_env.sh      # one-time: mamba env + Flutter SDK + Android SDK into .tooling/
source scripts/activate.sh     # every shell: puts flutter/dart/java/sdkmanager on PATH
flutter doctor                 # Flutter + Android toolchain should be green
```

Everything project-specific is isolated:
- `environment.yml` → mamba env `zany-test-prep` (JDK 17 + python content tooling).
- `.tooling/` (git-ignored) → Flutter SDK + Android SDK, installed by `scripts/setup_env.sh`.

## Key commands (once content/app exist)

```bash
python tools/validate_content.py          # validate offline content
python tools/generate_content.py          # regenerate content bank
python tools/build_bundle.py              # build assets/content/sat.bundle.json
flutter test                              # unit + widget tests
flutter test integration_test             # integration test
flutter build apk --release               # release APK -> build/app/outputs/flutter-apk/
```

## Notes / decisions

- Content is generated by deterministic Python generators that guarantee answer
  correctness (math computed; RW conventions rule-based; reading from an authored
  original-passage pool). Canonical content lives in `content/exams/sat/`; the app
  loads a single bundled `assets/content/sat.bundle.json`.
- Validator is Python (`tools/validate_content.py`, uses jsonschema) and also runs in CI;
  Dart tests additionally prove the app can parse the bundle.
