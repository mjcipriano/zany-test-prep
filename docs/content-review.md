# Content review — progress tracker

A section-at-a-time audit + improvement of the SAT content. Work one section per
pass, commit, then move to the next so progress survives interruptions. **This
file is the source of truth for where we are.**

Content is generated by Python (`tools/content_gen/`), so "fixing a section"
means fixing its generator(s) + teaching, then regenerating:

```bash
source scripts/activate.sh
python tools/generate_content.py        # regenerate content/exams/sat/**
python tools/validate_content.py        # must pass (no errors); minimize warnings
python tools/build_bundle.py            # rebuild assets/content/sat.bundle.json
flutter test                            # keep green
```

A "section" = one skill across its easy/medium/hard tiers.

## Definition of done (per section)

1. **Difficulty ramps** — easy → medium → hard get *substantially* harder
   (bigger/uglier numbers, more steps, more abstraction), not just relabeled.
2. **No answer tells** — correct vs. wrong answers show no pattern: not always the
   same position (validator enforces), not always longest, and **not always the
   same first word / wording** (validator: `answer_wording_flags`). Distractors are
   plausible.
3. **Variation + trickier distractors** — multiple prompt phrasings and answer
   wordings; harder tiers use common-mistake distractors (sign errors, off-by-one,
   half/double, swapped operations, etc.).
4. **Coverage** — add missing question types/sub-skills the real SAT tests.
5. **Figures** — add an image/figure where the SAT would show one (geometry,
   scatterplots, some data analysis). Needs the figure infra (see Phase 0).
6. **Mini-lesson** — a 2–5 screen teaching intro that *teaches* how to solve, not
   just quizzes (the lesson's `teaching_card.screens`).
